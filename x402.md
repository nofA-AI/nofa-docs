# x402 XRPL Payment-Gated API

> Base URL (switch by environment):
>
> | Environment | BASE_URL |
> |-------------|----------|
> | Development | `https://api-dev.reclaw.xyz` |


All examples below use the variable `BASE_URL`. Replace it with the URL for your target environment.

---

## Overview

NOFA uses the [x402 protocol](https://xrpl-x402.t54.ai/docs/quickstart) to gate backtest and dry-run trading APIs behind XRPL payments.
When a client hits a protected endpoint, the server returns **HTTP 402 Payment Required**. After signing an XRPL Payment transaction, the client retries the request and gains access.

### Protected Endpoints

| Method | Path | Description |
|--------|------|-------------|
| POST | `/api/v1/x402/backtest` | Paid backtest |
| POST | `/api/v1/x402/dry-run` | Paid dry-run trading |

---

## Full Flow

```
Client                          Server                      Facilitator
  |                               |                              |
  |  1. POST /api/v1/x402/backtest                               |
  |  ------------------------------>                              |
  |                               |                              |
  |  2. HTTP 402 + PAYMENT-REQUIRED header (base64 JSON)         |
  |  <------------------------------                             |
  |                               |                              |
  |  3. Parse PAYMENT-REQUIRED,                                  |
  |     build XRPL Payment tx,                                   |
  |     bind invoiceId, sign                                     |
  |                               |                              |
  |  4. POST /api/v1/x402/backtest                               |
  |     + PAYMENT-SIGNATURE header (base64 JSON)                 |
  |  ------------------------------>                              |
  |                               |  5. verify + settle          |
  |                               |  ------------------------->  |
  |                               |  <-------------------------  |
  |                               |                              |
  |  6. HTTP 200 + business response                             |
  |     + PAYMENT-RESPONSE header                                |
  |  <------------------------------                             |
```

## Step 1: Send Request, Receive 402

```bash
BASE_URL=https://api-dev.reclaw.xyz

curl -s -w "\nHTTP_STATUS: %{http_code}\n" \
  -X POST "${BASE_URL}/api/v1/x402/backtest" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <your-token>" \
  -d '{}'
```

### 402 Response Example

```http
HTTP/1.1 402 Payment Required
PAYMENT-REQUIRED: eyJhY2NlcHRzIjpbey...base64...}
Content-Type: application/json

{
  "error": "X-PAYMENT header is required",
  "accepts": [
    {
      "scheme": "exact",
      "network": "xrpl:1",
      "asset": "XRP",
      "payTo": "rReceiverWalletAddress",
      "amount": "1000",
      "maxTimeoutSeconds": 600,
      "extra": {
        "invoiceId": "unique-invoice-uuid",
        "sourceTag": 804681468
      }
    }
  ]
}
```

### Key Fields

| Field | Description |
|-------|-------------|
| `scheme` | Always `"exact"` — XRPL exact payment scheme |
| `network` | CAIP-2 network ID: `xrpl:0` = mainnet, `xrpl:1` = testnet, `xrpl:2` = devnet |
| `asset` | `"XRP"` or IOU currency code |
| `payTo` | Receiving XRPL address |
| `amount` | Amount in drops for XRP (1 XRP = 1,000,000 drops) |
| `maxTimeoutSeconds` | Payment validity window (seconds) |
| `extra.invoiceId` | Unique invoice ID for anti-replay binding |
| `extra.sourceTag` | XRPL SourceTag, default `804681468` |

## Step 2: Build and Sign an XRPL Payment Transaction

The client must build an XRPL Payment transaction based on the `accepts` data from the 402 response.

### Invoice Binding (Anti-Replay)

The transaction **must** bind the `invoiceId`. Two methods are supported (recommended to use both):

| Method | Field | Value |
|--------|-------|-------|
| **Memos** | `MemoData` | `HEX(UTF-8(invoiceId))` |
| **InvoiceID** | `InvoiceID` | Uppercase hex of `SHA256(invoiceId)` |

### Python Example: Build Transaction Manually

```python
import hashlib
import math

from xrpl.clients import JsonRpcClient
from xrpl.ledger import get_latest_validated_ledger_sequence
from xrpl.models.transactions import Memo, Payment
from xrpl.transaction import autofill, sign
from xrpl.wallet import Wallet

# ---- Configuration ----
TESTNET_RPC = "https://s.altnet.rippletest.net:51234"
client = JsonRpcClient(TESTNET_RPC)

# Parsed from 402 response
pay_to = "rReceiverWalletAddress"
amount_drops = "1000"
invoice_id = "unique-invoice-uuid"
max_timeout_seconds = 600
source_tag = 804681468

# Your wallet (testnet faucet: https://faucet.altnet.rippletest.net)
wallet = Wallet.from_seed("sYourTestnetSeed")

# ---- Invoice binding ----
memo_hex = invoice_id.encode("utf-8").hex().upper()
invoice_id_hash = hashlib.sha256(invoice_id.encode("utf-8")).hexdigest().upper()

# ---- LastLedgerSequence (expiry protection) ----
current_ledger = get_latest_validated_ledger_sequence(client)
max_ledger_delta = int(math.ceil(max_timeout_seconds / 5.0) + 2)
last_ledger_seq = current_ledger + max_ledger_delta

# ---- Build Payment transaction ----
payment = Payment(
    account=wallet.classic_address,
    destination=pay_to,
    amount=amount_drops,              # XRP drops
    source_tag=source_tag,
    last_ledger_sequence=last_ledger_seq,
    memos=[Memo(memo_data=memo_hex)],  # Memos binding
    invoice_id=invoice_id_hash,        # InvoiceID binding
)

# ---- Autofill + sign ----
filled = autofill(payment, client)
signed = sign(filled, wallet)
signed_tx_blob = signed.blob()

print(f"Signed TX Blob: {signed_tx_blob}")
```

## Step 3: Build the PAYMENT-SIGNATURE Header

Package the signed transaction as base64-encoded JSON in the `PAYMENT-SIGNATURE` request header.

### Header Structure

```json
{
  "x402Version": 2,
  "accepted": {
    "scheme": "exact",
    "network": "xrpl:1",
    "asset": "XRP",
    "payTo": "rReceiverWalletAddress",
    "amount": "1000",
    "maxTimeoutSeconds": 600,
    "extra": {
      "sourceTag": 804681468,
      "invoiceId": "unique-invoice-uuid"
    }
  },
  "payload": {
    "signedTxBlob": "120000228...(hex encoded signed tx)...",
    "invoiceId": "unique-invoice-uuid"
  }
}
```

### Python Example: Encode the Header

```python
import base64
import json

payment_signature = {
    "x402Version": 2,
    "accepted": {
        "scheme": "exact",
        "network": "xrpl:1",
        "asset": "XRP",
        "payTo": pay_to,
        "amount": amount_drops,
        "maxTimeoutSeconds": max_timeout_seconds,
        "extra": {
            "sourceTag": source_tag,
            "invoiceId": invoice_id,
        },
    },
    "payload": {
        "signedTxBlob": signed_tx_blob,
        "invoiceId": invoice_id,
    },
}

header_value = base64.b64encode(
    json.dumps(payment_signature, separators=(",", ":"), sort_keys=True).encode()
).decode()

print(f"PAYMENT-SIGNATURE: {header_value}")
```

### Option B: Manual Flow

Full control over each step. Useful as a reference for non-Python clients.

```python
import base64
import hashlib
import json
import math

import requests
from xrpl.clients import JsonRpcClient
from xrpl.ledger import get_latest_validated_ledger_sequence
from xrpl.models.transactions import Memo, Payment
from xrpl.transaction import autofill, sign
from xrpl.wallet import Wallet

BASE_URL = "https://api-dev.reclaw.xyz"
TESTNET_RPC = "https://s.altnet.rippletest.net:51234"
AUTH_HEADER = {"Authorization": "Bearer <your-token>"}

wallet = Wallet.from_seed("sYourTestnetSeed")
xrpl_client = JsonRpcClient(TESTNET_RPC)

backtest_payload = {
    "strategy": {
        "type": "STRATEGY_TREE",
        "name": "RSI Oversold Strategy",
        "riskManagement": {
            "type": "RISK_MANAGEMENT",
            "name": "Global Risk",
            "scope": "Per Position",
            "stopLoss": {"mode": "PCT", "value": 0.03},
            "takeProfit": {"mode": "PCT", "value": 0.06},
        },
        "mainDecision": {
            "type": "IF_ELSE_BLOCK",
            "name": "RSI Check",
            "conditionType": "Compare",
            "conditions": [
                {
                    "type": "CONDITION_ITEM",
                    "indicator": "RSI",
                    "period": 14,
                    "symbol": "BTC/USDT:USDT",
                    "operator": "Less Than",
                    "value": 30,
                }
            ],
            "thenAction": [
                {
                    "type": "ACTION_BLOCK",
                    "name": "Long BTC",
                    "symbol": "BTC/USDT:USDT",
                    "direction": "LONG",
                    "allocate": {
                        "type": "ALLOCATE_CONFIG",
                        "mode": "WEIGHT",
                        "value": 50,
                    },
                    "leverage": 1,
                }
            ],
            "elseAction": "NO ACTION",
        },
    },
    "capital": 10000,
    "start_time": "2025-12-01T00:00:00Z",
    "end_time": "2025-12-31T00:00:00Z",
    "timeframe": "1h",
    "slippage": 0.001,
    "transaction_fee": 0.0005,
}

# ---- Step 1: Send request, receive 402 ----
resp = requests.post(
    f"{BASE_URL}/api/v1/x402/backtest",
    json=backtest_payload,
    headers=AUTH_HEADER,
)

if resp.status_code != 402:
    print(f"Unexpected status: {resp.status_code}")
    print(resp.json())
    exit(1)

body = resp.json()
accepts = body["accepts"]
reqs = accepts[0]  # Select the first payment option

pay_to = reqs["payTo"]
amount = reqs["amount"]
network = reqs["network"]
max_timeout = reqs["maxTimeoutSeconds"]
invoice_id = reqs["extra"]["invoiceId"]
source_tag = reqs["extra"].get("sourceTag", 804681468)

print(f"Payment required: {amount} drops to {pay_to} (invoice: {invoice_id})")

# ---- Step 2: Build XRPL Payment transaction ----
memo_hex = invoice_id.encode("utf-8").hex().upper()
invoice_id_hash = hashlib.sha256(invoice_id.encode("utf-8")).hexdigest().upper()

current_ledger = get_latest_validated_ledger_sequence(xrpl_client)
last_ledger_seq = current_ledger + int(math.ceil(max_timeout / 5.0) + 2)

payment_tx = Payment(
    account=wallet.classic_address,
    destination=pay_to,
    amount=str(int(amount)),
    source_tag=source_tag,
    last_ledger_sequence=last_ledger_seq,
    memos=[Memo(memo_data=memo_hex)],
    invoice_id=invoice_id_hash,
)

filled = autofill(payment_tx, xrpl_client)
signed_tx = sign(filled, wallet)
signed_blob = signed_tx.blob()

# ---- Step 3: Build PAYMENT-SIGNATURE header ----
signature_payload = {
    "x402Version": 2,
    "accepted": {
        "scheme": reqs["scheme"],
        "network": network,
        "asset": reqs["asset"],
        "payTo": pay_to,
        "amount": amount,
        "maxTimeoutSeconds": max_timeout,
        "extra": {
            "sourceTag": source_tag,
            "invoiceId": invoice_id,
        },
    },
    "payload": {
        "signedTxBlob": signed_blob,
        "invoiceId": invoice_id,
    },
}

payment_header = base64.b64encode(
    json.dumps(signature_payload, separators=(",", ":"), sort_keys=True).encode()
).decode()

# ---- Step 4: Retry with signed payment ----
resp = requests.post(
    f"{BASE_URL}/api/v1/x402/backtest",
    json=backtest_payload,
    headers={
        **AUTH_HEADER,
        "PAYMENT-SIGNATURE": payment_header,
    },
)

print(f"Status: {resp.status_code}")
print(json.dumps(resp.json(), indent=2))

# Parse payment result
if "PAYMENT-RESPONSE" in resp.headers:
    pr = json.loads(base64.b64decode(resp.headers["PAYMENT-RESPONSE"]))
    print(f"TX Hash: {pr.get('transaction')}")
    print(f"Payer: {pr.get('payer')}")
```

## Error Handling

### Common HTTP Status Codes

| Status | Meaning |
|--------|---------|
| `200` | Payment successful, business data returned |
| `401` | Unauthorized (missing or invalid Authorization header) |
| `402` | Payment required (first request or invalid signature) |
| `422` | Validation error (malformed strategy JSON, etc.) |

### Facilitator Verification Errors

When the PAYMENT-SIGNATURE is rejected by the facilitator, the server returns 402 with an error message:

| Error Code | Description |
|------------|-------------|
| `invalid_x402_version` | x402Version is not 2 |
| `unsupported_scheme` | scheme is not "exact" |
| `invalid_network` | Network identifier mismatch |
| `destination_mismatch` | Transaction destination does not match payTo |
| `amount_mismatch` | Payment amount does not match |
| `source_tag_mismatch` | SourceTag does not match |
| `invoice_binding_missing` | Missing invoiceId binding |
| `invoice_binding_mismatch` | invoiceId binding value mismatch |
| `missing_last_ledger_sequence` | Transaction missing expiry protection |
| `settle_failed_onchain` | On-chain settlement failed |
